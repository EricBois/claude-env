2. claude-env launcher

Path: ~/bin/claude-env. Creates two sibling git worktrees and a tmux session with three named windows. Survives Ghostty restarts via tmux attach.

Install summary

  • Script: ~/bin/claude-env (executable)
  • Presets: ~/.config/claude-env/presets
  • ~/bin added to PATH in .bashrc
  • Dependencies: git, tmux, claude, optional difit

Usage

CommandEffect
claude-envPick preset interactively (numbered menu).
claude-env REPOUse preset named REPO.
claude-env ~/projects/fooUse explicit repo path.
claude-env --listPrint all presets.
claude-env REPO --startAuto-run claude in every tab.
claude-env REPO --askPrompt per tab before starting claude.
claude-env REPO --add bugfixWorktree+window bugfix on branch bugfix.
claude-env REPO --add worktree1,worktree2Classic two-worktree layout via comma list.
claude-env REPO --add review:origin/mainWorktree review tracking branch origin/main.
claude-env REPO --add a --add b,cRepeatable flag, comma list works in either.

--add NAME[:BRANCH] semantics

  • Worktree dir: <repo>-NAME (sibling of repo).
  • Branch: BRANCH if supplied after colon, else NAME itself.
  • If branch exists, worktree checks it out. If not, created with worktree add -b.
  • Idempotent: re-running with the same --add reuses the worktree and skips creating a duplicate tmux window.
  • Works on first launch or when reattaching an existing session — extras get appended either way.

Behaviour

  • Default mode = manual. Each tab opens login shell only. Type claude when ready.
  • Worktrees created on first run, reused after. Idempotent.
  • tmux session named claude-<repo-name>. Re-running attaches instead of duplicating.
  • Default layout: just the main window (repo root). All extra workspaces are opt-in via --add.
  • Cleanup is intentionally manual — see Troubleshooting §7.
  • Add tmux F-key bindings for new windows if you want fast jumps — edit ~/.tmux.conf to add e.g. bind -n F4 select-window -t bugfix.

# ~/.tmux.conf

# --- Direct window switching by NAME (Alt + number) ---
bind -n M-1 select-window -t main
bind -n M-2 select-window -t worktree1
bind -n M-3 select-window -t worktree2

# --- F-key fallback (TUIs don't eat F-keys) ---
bind -n F1 select-window -t main
bind -n F2 select-window -t worktree1
bind -n F3 select-window -t worktree2

# --- Ctrl-Shift arrows for prev/next ---
bind -n C-S-Left  previous-window
bind -n C-S-Right next-window

# --- Reload config ---
bind r source-file ~/.tmux.conf \; display "tmux config reloaded"

# --- difit: diff viewer popup ---
bind d display-popup -d "#{pane_current_path}" -w 90% -h 85% -E \
  "difit . ; read -n1 -s -p 'press any key to close'"
bind D detach-client
bind C-d run-shell "cd #{pane_current_path} && difit . >/dev/null 2>&1 &"

5. difit — diff review viewer

difit is a CLI tool that serves a GitHub-style diff viewer in your browser. Review Claude Code's changes, leave comments, export comments as an AI prompt back to Claude. Repo: github.com/yoshiko-pg/difit.

Install

npm install -g difit
# or one-shot:  npx difit

Common invocations

CommandShows
difitHEAD commit
difit .All uncommitted changes (working tree vs HEAD)
difit stagedOnly staged changes
difit . mainWorking tree vs main branch
difit HEAD~3Compare HEAD with HEAD~3
difit --port 5001Custom port (use if conflicts)
difit --pr <url>Review a GitHub PR

Integration with claude-env

  • From any tmux window: Ctrl+b d opens difit . in a popup, serving the current pane's cwd.
  • Each window has the correct worktree cwd → review the right diff with zero typing.
  • difit picks a free port automatically — safe to run in worktree1 and worktree2 simultaneously.
  • Press any key to close the popup; the browser tab stays usable until you close it.

6. Daily workflow

  1. Launch Ghostty.
  2. claude-env REPO --add worktree1,worktree2 — main window + two worktree windows.
  3. tmux session attaches. You land on window main.
  4. F2 → jump to worktree1. Type claude.
  5. Describe the task. Claude edits files in the worktree1 worktree.
  6. Ctrl+b d → diff viewer popup, review changes in browser.
  7. Leave inline comments in difit, export as prompt, paste back into Claude.
  8. Commit when satisfied. F3 to start a parallel task in worktree2 without context-switching the main agent.
  9. Need another isolated branch? From outside the session: claude-env REPO --add hotfix appends a new window to the running session.
  10. Ctrl+b D detaches; close Ghostty. Reopen later: claude-env REPO reattaches the same session.

The F-key bindings in ~/.tmux.conf map F1/F2/F3 to window names main / worktree1 / worktree2. If you use different worktree names, either rename them in ~/.tmux.conf or rely on Ctrl+b w (window picker).

7. Remote access (phone / laptop / anywhere)

The tmux session lives on the host running claude-env, so any device that can SSH in can reattach the same Claude Code session — including an Android phone via Termux.

SSH vs mosh

TransportNotes
sshWorks everywhere. TCP — drops on Wi-Fi → LTE handoffs, suspend/resume, or weak signal.
moshUDP-based. Survives roaming, sleep/wake, and flaky mobile connections. Recommended for phone/tablet use.

Tailscale for zero-config networking

Install Tailscale on both ends to get a private network between your devices — no port forwarding, no public IP, no DNS setup. The host appears as a stable hostname on every client.

From Termux on Android

# one-time
pkg install openssh mosh
# also install Tailscale from F-Droid or Play Store, then `tailscale up`

# connect
mosh <tailscale-host>             # preferred on mobile
# or
ssh <tailscale-host>

# reattach the workspace
claude-env REPO
Because the tmux session is persistent, closing Termux, locking the phone, or switching networks does not lose state. Reconnect → claude-env REPO → same session.

8. Troubleshooting

“command not found: claude-env”

source ~/.bashrc          # PATH not loaded in current shell
# or open a fresh terminal

“no presets defined” but file has entries

Fixed: parser tolerates missing trailing newline. If still broken, ensure each line matches name = path (spaces around = ok, comments with # ok).

Alt shortcuts don't work inside Claude

Claude Code's TUI eats Alt+number for its own input editing. Use F1/F2/F3 instead — F-keys reliably pass through to tmux.

tmux status bar truncates window names

Already widened in ~/.tmux.conf (status-left-length 30). If still truncated, increase Ghostty window width or shrink status-right.

difit port already in use

difit . --port 5001

Kill a stuck session

tmux ls
tmux kill-session -t claude-myrepo

Manual cleanup

By design, claude-env never removes worktrees. Tear down by hand when you're done with a feature:

# 1. Detach from tmux if still attached:
#    Ctrl+b D

# 2. Inspect what's there
git -C ~/projects/myrepo worktree list

# 3. Remove one worktree (clean working tree required)
git -C ~/projects/myrepo worktree remove ../myrepo-worktree1

# 4. Force-remove if dirty
git -C ~/projects/myrepo worktree remove ../myrepo-worktree1 --force

# 5. Delete the branch if you no longer want it
git -C ~/projects/myrepo branch -D worktree1

# 6. Prune stale worktree metadata after manual rm
git -C ~/projects/myrepo worktree prune

# 7. Optionally drop the tmux session
tmux kill-session -t claude-myrepo

# Re-running `claude-env REPO --add worktree1,...` will recreate the worktree.
Recap. One command per project. main window by default, extra worktree windows via --add (comma-separated or repeated). In-place diff review via difit popup. Manual cleanup with git worktree remove. Add new projects by editing the presets file.
